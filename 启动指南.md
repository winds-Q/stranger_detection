# 笔记本摄像头陌生人检测报警系统 - 程序启动指南

---

## 一、系统简介

本系统利用笔记本自带摄像头，实时检测出现在镜头前的人员。若检测到陌生人（非注册熟人），系统将自动发送邮件报警至指定邮箱，附带陌生人截图附件。

系统提供两种运行方式：

| 方式 | 说明 |
|------|------|
| **命令行模式** | `python main.py`，日志输出到控制台 |
| **Web UI 模式** | `python app.py`，浏览器可视化操作 |

**核心流程：**

```
摄像头采集帧 → 人脸检测 → 人脸编码 → 与熟人库比对 → 陌生人？→ 发送邮件报警
```

## 二、环境要求

| 组件 | 要求 |
|------|------|
| 操作系统 | Windows / Linux / macOS |
| Python | 3.10 或更高版本 |
| 摄像头 | 笔记本内置摄像头或 USB 外接摄像头 |
| 网络 | 需要网络连接（用于 SMTP 邮件发送） |
| 浏览器 | Web UI 模式需要现代浏览器 |

### 核心依赖库

| 库 | 版本 | 用途 |
|----|------|------|
| Python | >= 3.10 | 运行环境 |
| opencv-python | >= 4.8 | 摄像头采集与图像处理 |
| face-recognition | >= 1.3.0 | 人脸检测、编码、比对 |
| dlib | >= 19.24 | face_recognition 底层依赖（自动安装） |
| pyyaml | >= 6.0 | 配置文件解析 |
| flask | >= 3.0 | Web UI 服务框架 |
| smtplib / email | 内置 | 邮件发送（Python 标准库） |

## 三、快速启动

### 3.1 安装依赖

```bash
cd stranger_detection
pip install -r requirements.txt
```

> **Windows 用户注意：** `face-recognition` 依赖 `dlib`，Windows 下安装可能需要 CMake 和 Visual Studio Build Tools。
> 如安装失败，可先下载 dlib 预编译 wheel：
> ```bash
> pip install https://github.com/z-mahmud22/Dlib_Windows_Python3.x/raw/main/dlib-19.24.99-cp310-cp310-win_amd64.whl
> pip install face-recognition opencv-python pyyaml flask
> ```

### 3.2 配置 SMTP 邮箱

编辑项目根目录下的 `config.yaml`，修改邮件配置：

```yaml
alert:
  enabled: true
  cooldown_seconds: 180
  smtp_server: "smtp.gmail.com"
  smtp_port: 587
  sender_email: "your_email@gmail.com"          # 改为你的邮箱
  # 授权码仅通过环境变量 STRANGER_DETECTION_SMTP_PASSWORD 提供
  receiver_emails:                                # 可配置一个或多个收件邮箱
    - "first@example.com"
    - "second@example.com"
```

> **推荐使用环境变量**（更安全），避免在配置文件中写入密码：
> ```bash
> # Windows PowerShell:
> $env:STRANGER_DETECTION_SMTP_PASSWORD = "你的SMTP授权码"
>
> # Linux/macOS:
> export STRANGER_DETECTION_SMTP_PASSWORD="你的SMTP授权码"
> ```

**常用 SMTP 服务器参考：**

| 邮箱 | SMTP 服务器 | 端口 |
|------|-------------|------|
| Gmail | smtp.gmail.com | 587 |
| QQ邮箱 | smtp.qq.com | 587 |
| 163邮箱 | smtp.163.com | 465 |
| Outlook | smtp.office365.com | 587 |

> 需要使用 SMTP 授权码而非登录密码（在邮箱设置中生成）。

### 3.3 注册熟人照片

**方式一：直接复制文件**

将熟人照片放入 `known_faces/` 目录：

```
known_faces/
├── alice.jpg          # 文件名即为显示名称
├── alice_side.jpg     # 同一人可放多张不同角度/光照的照片
├── bob.jpg
└── bob_glasses.jpg
```

**方式二：通过 Web UI 上传**（推荐，启动 Web UI 后操作）

先填写“熟人姓名”，再选择一张或多张属于该熟人的照片。系统会为样本生成唯一文件名，识别结果、画面标注和重要日志仍统一显示填写的姓名。

**建议：**
- 每人放 5~10 张照片（正面、侧面、不同光照）
- 每张照片只包含一个人脸
- 支持 jpg / png / bmp 格式
- 文件名即为日志中显示的名称

### 3.4 启动程序

#### 方式一：命令行模式（cli）

```bash
cd stranger_detection/src
python main.py
```

程序启动后将：
1. 输出日志到控制台和 `logs/` 目录
2. 打开摄像头开始实时检测
3. 检测到陌生人时自动发送邮件报警

按 **Ctrl+C** 可安全退出。

#### 方式二：Web UI 模式（推荐）

```bash
cd stranger_detection
python run_web.py
```

浏览器访问 **http://localhost:5050**

Web UI 提供以下功能：

| 功能 | 说明 |
|------|------|
| **启动/停止检测** | 点击按钮在后台线程启动或停止检测，不影响 Web 服务 |
| **上传熟人照片** | 点击或拖拽上传照片到 `known_faces/`，支持批量上传 |
| **管理熟人列表** | 查看已注册熟人，hover 显示删除按钮 |
| **实时日志监控** | 右侧面板实时显示系统日志，按级别着色（ERROR 红 / WARNING 黄） |
| **状态指示** | 顶部显示当前运行状态，绿点脉冲=运行中，灰点=已停止 |
| **健康状态面板** | 显示摄像头状态、检测 FPS、熟人数量、今日告警、邮件配置及队列状态 |
| **摄像头管理** | 扫描并选择笔记本内置或 USB/Type-C 外接摄像头 |
| **告警历史** | 查看 SQLite 中保存的告警记录和截图，可标记已处理或删除 |
| **测试邮件** | 使用当前 SMTP 参数发送测试邮件，不显示或保存授权码 |

实时日志仅显示检测、识别、报警和异常等重要事件，不显示 `/api/status` 等浏览器轮询访问记录。识别到熟人时会输出熟人姓名；为避免刷屏，同一熟人 30 秒内只记录一次。

> **注意：** Web UI 启动后，检测线程、熟人照片管理和 Web 服务在同一个进程中运行，关闭浏览器不会停止检测（需点击"停止检测"按钮或 Ctrl+C 退出程序）。
>
> Web 服务默认只监听本机 `127.0.0.1`。如需让局域网设备访问，可设置
> `STRANGER_DETECTION_WEB_HOST=0.0.0.0`，但建议同时增加防火墙和访问控制。
>
> `run_web.py` 使用 Waitress 正式服务器，并在启动前检查端口，防止重复运行两个 Web 实例。`src/web/app.py` 仅保留为开发调试入口。

## 四、配置项说明 (`config.yaml`)

```yaml
camera:
  device_id: 0          # 摄像头设备 ID（0 为笔记本自带摄像头，1/2 为外接）
  width: 640            # 采集分辨率宽
  height: 480           # 采集分辨率高

recognition:
  tolerance: 0.5        # 人脸匹配阈值（0.4~0.6），越小越严格，越小越易误报
  model: "hog"          # hog=CPU快速 | cnn=GPU更准但较慢

alert:
  enabled: true         # 是否启用邮件报警
  cooldown_seconds: 180 # 同一陌生人三分钟内不重复报警，不同陌生人互不影响
  smtp_server: "smtp.gmail.com"
  smtp_port: 587
  sender_email: "your_email@gmail.com"
  # 授权码仅通过环境变量 STRANGER_DETECTION_SMTP_PASSWORD 提供
  receiver_emails:
    - "first@example.com"
    - "second@example.com"

snapshot:
  save_to: "./snapshots/"
  max_snapshots: 100    # 最多保留截图数量

database:
  path: "./data/alerts.db"  # 告警事件 SQLite 数据库
  face_cache_path: "./data/face_encodings.db" # 熟人人脸编码缓存

retention:
  snapshots_days: 7          # 截图保留天数
  logs_days: 14              # 日志保留天数
  events_days: 30            # 告警历史保留天数
  cleanup_interval_hours: 12 # 自动清理间隔

face_upload_validation:
  min_face_size: 80          # 人脸区域最小边长（像素）
  min_blur_score: 40         # 人脸区域清晰度下限，越大越严格
  min_brightness: 35         # 最低平均亮度
  max_brightness: 225        # 最高平均亮度
  duplicate_image_difference: 2.0 # 重复图片差异阈值，越小越严格

web:
  startup_timeout_seconds: 30 # 摄像头、模型及熟人样本初始化等待时间
```

### 环境变量

| 变量名 | 说明 |
|--------|------|
| `STRANGER_DETECTION_CONFIG` | 自定义配置文件路径，覆盖默认 `config.yaml` |
| `STRANGER_DETECTION_SMTP_PASSWORD` | SMTP 密码/授权码；唯一受支持的授权码来源，不会通过 Web 返回或写入 YAML |

> 修改环境变量后需要重启程序。旧版 `config.yaml` 中如果存在 `sender_password`，当前版本会忽略该字段；在 Web 页面保存一次配置后会自动移除它。

### 熟人照片上传质量检查

通过 Web 上传熟人照片时，系统会在保存前执行以下检查：

- 图片中必须恰好有一张人脸；
- 人脸不能过小；
- 照片不能过暗、过亮或明显模糊；
- 必须能够提取有效的人脸特征；
- 不能与熟人目录中已有的人脸照片重复。

检查失败时页面会显示中文原因，文件不会写入熟人目录。直接复制到 `known_faces/` 的文件不会经过 Web 上传检查，因此仍建议优先使用 Web 上传。

### 告警历史与自动清理

陌生人事件会写入 `data/alerts.db`。Web 页面可查看事件时间、临时身份、邮件发送结果和截图，并可标记已处理或删除。程序会根据 `retention` 配置定期清理过期截图、日志和数据库事件；数据库目录及表结构会在首次运行时自动创建。

## 七、常见问题

### Q1: 摄像头打不开？
- 检查 `device_id` 是否正确（通常笔记本自带摄像头为 0）
- 确认没有其他程序占用摄像头（关闭 Zoom/Teams 等）
- 尝试改为 `device_id: 1`

### Q2: 朋友被误判为陌生人（误报）？
- 调大 `tolerance` 值（如 0.55 → 0.6）
- 增加该朋友的熟人照片（不同角度/光照/表情）
- 每张熟人照片确保只包含一个人脸

### Q3: 陌生人未被检测到（漏报）？
- 调小 `tolerance` 值（如 0.5 → 0.4）
- 尝试将 `model` 改为 `"cnn"`（需 GPU 或稍慢）

### Q4: CPU 占用过高？
- 降低摄像头分辨率（`width: 320, height: 240`）
- 使用 `model: "hog"`（CPU 友好）

### Q5: 邮件发送失败？
- 确认 `sender_email` / `receiver_emails` 中的地址填写正确
- 确认使用的是 SMTP 授权码（非登录密码）
- Gmail 需要在账户设置中开启"两步验证"并生成"应用专用密码"
- 确认已在启动程序的同一终端设置 `STRANGER_DETECTION_SMTP_PASSWORD`
- 可先在 Web 邮件配置区域点击“发送测试邮件”进行验证

### Q6: dlib 安装失败 (Windows)？
- 安装 Visual Studio Build Tools (含 C++ 工作负载)
- 或使用预编译 wheel 包绕过编译

### Q7: Web UI 无法访问？
- 确认 `flask` 已安装：`pip install flask`
- 确认端口 5050 未被占用，可修改 `app.py` 中的 `port` 参数
- 确认启动目录为 `stranger_detection/src/web/`
